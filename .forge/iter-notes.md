## Decisions made this iter
- **`CorrelationFooterFormat = "\n(trace: {0})"`** — appended after the action badge on a new line (not inlined into the badge) so the operator can visually parse "action" vs "trace" without confusing them; parentheses + lowercase "trace:" is the convention already used in many other operator-facing tools.
- **`ReplayCacheMaxSize = 10_000L`** — 10K callback ids × ~32-byte id × ~32-byte answer ≈ 640 KB worst case, bounded and recoverable on process restart. Larger caps would defeat the "bounded" purpose; smaller would risk evicting still-live entries in burst-alert scenarios with hundreds of agents.
- **`ReplayCacheTtl = TimeSpan.FromHours(1)`** — Telegram's `AnswerCallbackQuery` wire window is ~30s, but the dedup-service TTL horizon plus possible webhook redelivery extend beyond that. 1h covers any plausible in-process retry; ≥ 24h was rejected as inviting stale answers.
- **`MemoryCache` (process-local) instead of `IDistributedCache`** — the in-process layer correctly serves the in-process retry / Telegram-redelivery cases the evaluator complained about (unbounded growth). Cross-pod / restart sharing is Stage 4.3 scope and is explicitly called out in the field's XML doc as the next swap; the interface (`Set(key, text, opts)` + `TryGetValue(key, out text)`) maps 1:1 onto `IDistributedCache` so the swap is localised.
- **`IDisposable` on the handler** — DI container disposes singletons at root teardown; without `IDisposable`, the `MemoryCache`'s eviction timer keeps the cache rooted past intended lifetime in test fixtures.
- **Defensive empty-correlation-id branch in `BuildDecisionMessageText`** — kept the `IsNullOrEmpty` guard as cheap belt-and-suspenders even though `PendingQuestion.CorrelationId` is contractually non-empty (validated in the init setter). Removed the corresponding "empty correlation id" defensive test once it was clear the type contract makes the scenario unreachable.
